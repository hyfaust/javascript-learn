# 项目 05：排序算法可视化

## 项目目标

本项目通过可视化的方式展示四种经典排序算法的执行过程，帮助理解算法的内部工作原理。

**包含的算法：**
- 冒泡排序 (Bubble Sort)
- 选择排序 (Selection Sort)
- 插入排序 (Insertion Sort)
- 快速排序 (Quick Sort)

**核心技术：**
- 使用 `async/await` + `setTimeout` 实现动画帧控制
- 颜色编码展示算法每一步的状态
- 实时统计比较次数和交换次数
- 可调节排序速度和数组大小

## 如何运行

1. 直接在浏览器中打开 `index.html` 文件即可（双击或拖入浏览器）
2. 无需任何服务器或构建工具，纯静态页面

## 算法详解

### 1. 冒泡排序 (Bubble Sort)

**算法思想：** 重复遍历数组，每次比较相邻两个元素，顺序错误就交换。每轮遍历将最大元素"冒泡"到末尾。

**步骤演示：**
```
初始: [5, 3, 8, 1, 2]
第1轮: [3, 5, 1, 2, 8]    ← 8 冒泡到位
第2轮: [3, 1, 2, 5, 8]    ← 5 冒泡到位
第3轮: [1, 2, 3, 5, 8]    ← 3 冒泡到位
第4轮: [1, 2, 3, 5, 8]    ← 无交换，提前结束
```

**复杂度分析：**

| 指标 | 值 | 说明 |
|------|-----|------|
| 最好时间 | O(n) | 数组已有序，优化后只需一轮遍历 |
| 平均时间 | O(n²) | 随机排列 |
| 最坏时间 | O(n²) | 完全逆序 |
| 空间 | O(1) | 原地排序 |
| 稳定性 | ✅ 稳定 | 相等元素相对顺序不变 |

**代码优化：** 使用 `swapped` 标志位，如果一轮遍历中无交换则提前结束。

---

### 2. 选择排序 (Selection Sort)

**算法思想：** 将数组分为"已排序"和"未排序"两部分。每次从未排序部分找到最小元素，放到已排序部分末尾。

**步骤演示：**
```
初始: [5, 3, 8, 1, 2]    | 未排序: [5,3,8,1,2]
第1轮: [1, 3, 8, 5, 2]   | 找到最小值 1，与位置 0 交换
第2轮: [1, 2, 8, 5, 3]   | 找到最小值 2，与位置 1 交换
第3轮: [1, 2, 3, 5, 8]   | 找到最小值 3，与位置 2 交换
第4轮: [1, 2, 3, 5, 8]   | 找到最小值 5，与位置 3 交换（自身交换）
```

**复杂度分析：**

| 指标 | 值 | 说明 |
|------|-----|------|
| 最好时间 | O(n²) | 即使已有序仍需查找最小值 |
| 平均时间 | O(n²) | — |
| 最坏时间 | O(n²) | — |
| 空间 | O(1) | 原地排序 |
| 稳定性 | ❌ 不稳定 | 交换可能改变相等元素顺序 |

---

### 3. 插入排序 (Insertion Sort)

**算法思想：** 将数组分为"已排序"和"未排序"两部分。每次从未排序部分取第一个元素，在已排序部分中从后向前扫描，找到合适位置插入。类似整理手牌。

**步骤演示：**
```
初始: [5, 3, 8, 1, 2]
第1步: [3, 5, 8, 1, 2]   ← 将 3 插入 [5] 中
第2步: [3, 5, 8, 1, 2]   ← 8 已在正确位置
第3步: [1, 3, 5, 8, 2]   ← 将 1 插入 [3,5,8] 中
第4步: [1, 2, 3, 5, 8]   ← 将 2 插入 [1,3,5,8] 中
```

**复杂度分析：**

| 指标 | 值 | 说明 |
|------|-----|------|
| 最好时间 | O(n) | 数组已有序，每次只需比较一次 |
| 平均时间 | O(n²) | — |
| 最坏时间 | O(n²) | 完全逆序 |
| 空间 | O(1) | 原地排序 |
| 稳定性 | ✅ 稳定 | 相等元素不会交换 |

**适用场景：** 小规模数据或近似有序的数据时效率很高。

---

### 4. 快速排序 (Quick Sort)

**算法思想：** 分治法 (Divide and Conquer)：
1. 选择基准值 (Pivot)
2. 分区：小于 Pivot 的放左边，大于 Pivot 的放右边
3. 递归地对左右两部分重复上述过程

**步骤演示：**
```
初始: [5, 3, 8, 1, 2]
选择 Pivot = 2（最后一个元素）
分区:   [1] 2 [5, 3, 8]    ← 2 到位
左递归: [1]                  ← 单个元素，已有序
右递归: [5, 3, 8] 选 Pivot=8
       [5, 3] 8 []          ← 8 到位
       [3, 5]               ← 递归排序
结果:   [1, 2, 3, 5, 8]
```

**复杂度分析：**

| 指标 | 值 | 说明 |
|------|-----|------|
| 最好时间 | O(n log n) | 每次均匀分区 |
| 平均时间 | O(n log n) | 随机数据下表现优秀 |
| 最坏时间 | O(n²) | 每次分区极度不均衡（如已有序数组选尾元素为 Pivot） |
| 空间 | O(log n) | 递归调用栈深度 |
| 稳定性 | ❌ 不稳定 | 分区过程可能改变相等元素顺序 |

**注意事项：** 最坏情况可通过随机选择 Pivot 或使用"三数取中"策略避免。

## 算法对比总结

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 | 适用场景 |
|------|------|------|------|------|------|----------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ | 教学、小规模数据 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | ❌ | 交换次数要求少的场景 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ | 近似有序、小规模数据 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ | 大规模数据、通用排序 |

## 技术要点

### Async/Await 动画原理

```javascript
// 核心延迟函数
function sleep(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
}

// 在排序函数中使用
async function bubbleSort(arr) {
    for (let i = 0; i < n; i++) {
        // 修改柱子颜色/高度
        setBarState(j, 'comparing');

        // 暂停执行，让浏览器渲染当前状态
        await sleep(50);

        // 继续下一步...
    }
}
```

**工作流程：**
1. `await sleep(50)` 暂停当前 async 函数 50ms
2. 这 50ms 内浏览器执行渲染（柱子的新颜色/高度）
3. 50ms 后 Promise resolve，函数继续执行

这就是 JavaScript 事件循环 (Event Loop) 的妙用：虽然 JS 是单线程的，但 `await` 让出控制权给事件循环，使渲染任务得以执行。

### 速度控制

排序速度通过滑块实时调整，每次 `sleep()` 调用时读取当前滑块值：

```javascript
function getDelay() {
    return parseInt(DOM.speedRange.value, 10);
}

// 在算法循环中
await visualizeCompare(j, j + 1, getDelay()); // 每次读取最新速度
```

这样可以在排序过程中动态调整速度，无需重启。

### 钩子模式 (Hooks Pattern)

排序算法与可视化逻辑解耦，通过 hooks 对象传递回调：

```javascript
const hooks = {
    onCompare: (i, j) => { comparisonCount++; },
    onSwap: (i, j) => { swapCount++; },
    shouldStop: () => stopRequested,
};

// 算法内部调用
await visualizeCompare(j, j + 1, getDelay());
hooks.onCompare(j, j + 1); // 更新统计
```

## 挑战任务

### 🟢 简单：添加计数排序 (Counting Sort)

实现一个非比较排序算法——计数排序。

**要求：**
- 统计每个值出现的次数
- 根据计数重建有序数组
- 时间复杂度 O(n + k)，k 为值域范围

**提示：** 计数排序不通过比较交换实现，需要额外的计数数组。

### 🟡 中等：添加归并排序 (Merge Sort)

实现分治法的另一经典算法。

**要求：**
- 递归地将数组分为两半
- 分别排序后再合并
- 注意可视化合并过程（两个有序子数组合并）

**提示：** 归并排序是稳定排序，且最坏情况下仍为 O(n log n)。

### 🔴 困难：添加堆排序 (Heap Sort)

实现基于二叉堆的排序算法。

**要求：**
- 构建最大堆 (Max Heap)
- 依次取出堆顶元素并重新调整堆
- 可视化堆的树形结构（可选：在柱子图下方画一棵树）

**提示：** 堆排序的关键是 `heapify` 函数，它将一个子树调整为最大堆。时间复杂度始终为 O(n log n)。

### 🟣 超级挑战：双向冒泡排序 (Cocktail Shaker Sort)

冒泡排序的改进版，从左到右和从右到左交替遍历。

**优势：** 可以更快地将"乌龟"（较小的元素在数组末尾）移到正确位置。
