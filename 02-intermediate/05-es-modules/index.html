ES 模块 (ES Modules)

学习现代 JavaScript 模块化编程

一、模块导入与导出

1. 命名导出 (Named Exports)

使用 export 导出多个函数/变量

// math.js - 导出
export function add(a, b) { return a + b; }
export function multiply(a, b) { return a * b; }

// main.js - 导入
import { add, multiply } from './math.js';

2. 默认导出 (Default Export)

每个模块只能有一个默认导出

// config.js - 导出
export default { theme: "dark", lang: "zh" };

// main.js - 导入（可以自定义名称）
import config from './config.js';

3. 重命名导入

使用 as 关键字为导入的模块指定别名

import { add as sum, multiply as product } from './math.js';
// 现在可以使用 sum() 和 product() 而不是 add() 和 multiply()

二、工具函数模块

1. 日期格式化 (formatDate)

从 utils.js 模块导入日期格式化函数

import { formatDate } from './utils.js';
formatDate(new Date(), "YYYY-MM-DD");  // "2024-01-15"
formatDate(new Date(), "YYYY年MM月DD日");  // "2024年01月15日"

2. 生成唯一 ID (generateId)

生成简短的唯一标识符

import { generateId } from './utils.js';
generateId();  // "a3f9k2" (随机)

3. 防抖函数 (debounce)

延迟执行，避免频繁触发

import { debounce } from './utils.js';
const debouncedFn = debounce(() => console.log("执行!"), 500);
// 频繁调用时，只在最后一次调用后 500ms 执行
debouncedFn();  // 不会立即执行
debouncedFn();  // 重置计时器
debouncedFn();  // 再次重置

四、配置模块

config.js 设置

展示如何导入和使用配置对象

import config from './config.js';

console.log(config.lang);     // "zh"

五、模块注意事项

重要提示

📌 ES 模块关键点： 1. 必须在 <script type="module"> 中使用 2. 模块自动使用严格模式 ("use strict") 3. 每个模块有独立的作用域 4. 导入的路径必须包含扩展名 (.js) 5. 导入是只读的（live bindings） 6. 模块代码在页面加载时自动执行 ⚠️ 运行要求： 必须通过 HTTP 服务器运行（不能直接打开 file://） 因为浏览器的 CORS 安全策略限制。
